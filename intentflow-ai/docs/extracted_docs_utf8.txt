1. Executive Summary
IntentFlow AI is an intelligent, multimodal productivity platform that eliminates the gap between capturing ideas and executing tasks. 2. Problem Statement
Modern productivity tooling fails knowledge workers at three critical layers:
2.1  Input Friction
People capture ideas in bursts — voice memos while commuting, quick texts at their desk, screenshots from meetings. The friction between capture and actionable organization is so high that studies estimate over 60% of captured ideas never convert into tasks. Tools like Todoist and Notion require intentional, structured input that clashes with the speed of real-world thinking.
2.2  Context Fragmentation
A typical knowledge worker's commitments are scattered across Google Calendar, Gmail, Slack, Notion, and physical notebooks. There is no single source of truth. This fragmentation produces scheduling conflicts, missed follow-ups, and decision fatigue. Users spend more cognitive energy managing their systems than doing their actual work.
2.3  Automation Anxiety
Fully autonomous AI schedulers and task managers introduce a new failure mode: silent errors. Users have no visibility into why a meeting was moved, why a task priority changed, or why a follow-up was sent. This erodes trust. Without a human verification layer, autonomous systems remain unused or cause bigger problems than they solve.
Core insight: The market needs a system that is autonomous enough to eliminate manual work, but transparent enough that users trust it with important decisions. This is the design space IntentFlow AI occupies.

1. Development Flow Overview
1.1  The Vertical Slice Philosophy
We build IntentFlow AI using vertical slices: complete one feature from database to mobile UI before starting the next. This ensures you always have a working system and can test end-to-end at every stage.
Approach
Vertical Slice (✅ DO THIS)
Horizontal Layer (❌ AVOID)
Week 1
Build auth: DB + API + Mobile UI
Build entire database schema
Week 2
Build task creation: NLP + HITL + Display
Build all API endpoints
Week 3
Add voice input: UI + API integration
Build entire mobile UI
Result
Working app at every stage
Nothing works until week 12
1.2  Core Principles
Build → Test → Deploy → Iterate
After each feature, test it thoroughly and deploy to staging. Don't accumulate untested code.
One Working Feature > Five Half-Done Features
Finish what you start. A complete simple feature is more valuable than a complex incomplete one.
Test the Happy Path First, Edge Cases Later
Get the main flow working before handling every possible error scenario.
Database Changes Are Easy to Make Now, Hard Later
Spend extra time on database schema design upfront. Changing it later with production data is painful.

Day 2: Database Schema
Create users table (Supabase handles auth.users automatically)
CREATE TABLE public.users (
  id UUID PRIMARY KEY REFERENCES auth.users(id),
  email TEXT UNIQUE NOT NULL,
  full_name TEXT,
  avatar_url TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
Create tasks table
CREATE TABLE public.tasks (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  user_id UUID REFERENCES public.users(id) ON DELETE CASCADE,
  title TEXT NOT NULL,
  due_date DATE,
  due_time TIME,
  priority TEXT DEFAULT 'medium',
  status TEXT DEFAULT 'active',
  created_at TIMESTAMPTZ DEFAULT NOW()

5.2  The Case AGAINST Geolocation
Privacy Concerns
⚠️ Location data is extremely sensitive. Users are increasingly privacy-conscious. Many will reject location permissions entirely.
Requires explicit permission requests
Users may deny permission → feature becomes useless
App Store review scrutiny for location usage
GDPR compliance requirements for EU users
Must clearly explain WHY you need location
Battery Drain
Background location tracking is one of the biggest battery killers on mobile devices. Users will uninstall apps that drain battery.
Tracking Mode
Battery Impact
Acceptable?

None
✅ Always acceptable

✅ Acceptable

Low-Medium
⚠️ Maybe acceptable

❌ Usually unacceptable
Complexity vs Value
Geofencing requires background permissions (iOS is strict about this)
Testing location features is hard (simulators vs physical devices)
Edge cases: GPS drift, tunnel/indoor accuracy, permission revocation
Maintenance burden: OS updates break location APIs regularly
5.3  Recommendation: Phased Approach
Phase 1 (MVP): Skip Geolocation Entirely
🔴 Do NOT add geolocation in your first version. Focus on core productivity features. Validate product-market fit first.
Why skip it:
Adds significant complexity
Battery concerns will hurt app ratings
Privacy friction reduces signup conversion
You have enough to build already

const passport = require('passport');
const GoogleStrategy = require('passport-google-oauth20').Strategy;
const { createClient } = require('@supabase/supabase-js');
const supabase = createClient(
  process.env.SUPABASE_URL,
  process.env.SUPABASE_KEY
);
passport.use(new GoogleStrategy({
  clientID: process.env.GOOGLE_CLIENT_ID,
  clientSecret: process.env.GOOGLE_CLIENT_SECRET,
  callbackURL: '/auth/google/callback'
}, async (accessToken, refreshToken, profile, done) => {
  try {
    // Check if user exists
    const { data: existingUser } = await supabase
      .from('users')
      .select('*')
      .eq('oauth_id', profile.id)
      .single();
    
    if (existingUser) return done(null, existingUser);
    
    // Create new user
    const { data: newUser } = await supabase
      .from('users')
      .insert({
        oauth_id: profile.id,
        email: profile.emails[0].value,
        name: profile.displayName,
        profile_picture: profile.photos[0].value
      })
      .select()
      .single();
    
    return done(null, newUser);
  } catch (error) {
    return done(error, null);
  }
}));

const OpenAI = require('openai');
const openai = new OpenAI({
  apiKey: process.env.OPENAI_API_KEY
});
async function extractTaskIntent(inputText) {
  try {
    const completion = await openai.chat.completions.create({
      model: 'gpt-4',
      messages: [
        {
          role: 'system',
          content: `You are a task extraction assistant. Extract structured task data from user input.
Return JSON with this exact structure:
{
  "title": "short task description",
  "description": "longer details if provided",
  "due_date": "YYYY-MM-DD or null",
  "due_time": "HH:MM or null",
  "priority": "high" | "medium" | "low",
  "category": "work" | "personal" | "urgent" | "routine",
  "people": ["names mentioned"] or []
}
Rules:
- Extract relative dates: 'tomorrow' = tomorrow's date, 'next Friday' = calculate
- Default priority: medium
- If no date mentioned, due_date and due_time are null
- Title should be concise action-oriented phrase
`
        },
        {
          role: 'user',
          content: inputText
        }
      ],
      response_format: { type: 'json_object' },
      temperature: 0.3
    });
    const extracted = JSON.parse(completion.choices[0].message.content);
    
    // Calculate confidence score
    let confidence = 50;
    if (extracted.due_date) confidence += 25;
    if (extracted.due_time) confidence += 15;
    if (extracted.people && extracted.people.length > 0) confidence += 10;
    
    return {
      success: true,
      data: extracted,
      confidence: Math.min(confidence, 100)
    };
  } catch (error) {
    console.error('NLP Error:', error);
    return {
      success: false,
      error: error.message
    };
  }
}
module.exports = { extractTaskIntent };

import { useState } from 'react';
import axios from 'axios';
export default function TaskInput({ userId, onTaskCreated }) {
  const [input, setInput] = useState('');
  const [loading, setLoading] = useState(false);
  const [showPreview, setShowPreview] = useState(false);
  const [extractedData, setExtractedData] = useState(null);
  const handleExtract = async () => {
    if (!input.trim()) return;
    
    setLoading(true);
    try {
      const response = await axios.post('http://localhost:3001/api/nlp/extract', {
        input
      });
      
      if (response.data.success) {
        setExtractedData(response.data);
        setShowPreview(true);
      }
    } catch (error) {
      alert('Failed to extract task');
    } finally {
      setLoading(false);
    }
  };
  const handleApprove = async () => {
    try {
      await axios.post('http://localhost:3001/api/tasks', {
        user_id: userId,
        ...extractedData.data,
        confidence_score: extractedData.confidence
      });
      
      setInput('');
      setShowPreview(false);
      setExtractedData(null);
      onTaskCreated();
    } catch (error) {
      alert('Failed to create task');
    }
  };
  return (
    <div className="mb-8">
      <textarea
        value={input}
        onChange={(e) => setInput(e.target.value)}
        placeholder="Type anything... e.g., 'Call John tomorrow at 3pm'"
        className="w-full p-4 border-2 rounded-lg"
        rows={3}
      />
      <button
        onClick={handleExtract}
        disabled={loading}
        className="mt-2 bg-blue-600 text-white px-6 py-2 rounded-lg"
      >
        {loading ? 'Processing...' : 'Extract Task'}
      </button>
      {showPreview && extractedData && (
        <div className="mt-4 p-6 bg-gray-50 rounded-lg border-2">
          <h3 className="font-bold mb-4">Review Task</h3>
          <div className="space-y-2">
            <p><strong>Title:</strong> {extractedData.data.title}</p>
            <p><strong>Due:</strong> {extractedData.data.due_date} {extractedData.data.due_time}</p>
            <p><strong>Priority:</strong> {extractedData.data.priority}</p>
            <p><strong>Confidence:</strong> {extractedData.confidence}%</p>
          </div>
          <div className="mt-4 flex gap-4">
            <button
              onClick={handleApprove}
              className="bg-green-600 text-white px-6 py-2 rounded"
            >
              ✓ Approve
            </button>
            <button

4.2 NLP Extraction Service Code
// File: backend/services/nlp.js
// Prompt: Implement this complete NLP service:
 
const OpenAI = require('openai');
const { SYSTEM_PROMPT } = require('./nlp.prompt');
 
const openai = new OpenAI({ apiKey: process.env.OPENAI_API_KEY });
 
async function extractTaskIntent(inputText, userId) {
  const completion = await openai.chat.completions.create({
    model: process.env.NLP_MODEL || 'gpt-4',
    messages: [
      { role: 'system', content: SYSTEM_PROMPT },
      { role: 'user',   content: inputText }
    ],
    response_format: { type: 'json_object' },
    temperature: 0.3,
    max_tokens: 500,
  });
 
  const data = JSON.parse(completion.choices[0].message.content);
 
  // Confidence scoring algorithm
  let confidence = 50;
  if (data.due_date)              confidence += 20;
  if (data.due_time)              confidence += 15;
  if (data.people?.length > 0)    confidence += 10;
  if (data.priority !== "medium") confidence += 5;  // explicit priority
  if (data.category !== "work")   confidence += 5;  // explicit category
  confidence = Math.min(confidence, 100);
 
  // Auto-approve if above threshold
  const autoApprove = confidence >= (process.env.AUTO_APPROVE_THRESHOLD || 95);
 
  return { success: true, data, confidence, autoApprove };
}
 
module.exports = { extractTaskIntent };

// File: backend/services/hitl.confidence.js
// Prompt: Build confidence scoring with these rules:
 
function scoreConfidence(extracted) {
  let score = 50; // base score
 
  // Title quality (0–15 pts)
  if (extracted.title?.length > 3)   score += 10;
  if (extracted.title?.length > 10)  score += 5;
 
  // Temporal data (0–30 pts)
  if (extracted.due_date)            score += 20;
  if (extracted.due_time)            score += 10;
 
  // Context richness (0–15 pts)
  if (extracted.people?.length > 0)  score += 8;
  if (extracted.category !== "work") score += 4; // non-default = explicit
  if (extracted.priority !== "medium") score += 3;
 
  // Cap at 100
  return Math.min(score, 100);
}
 
// Threshold configuration:
// score >= 95  → AUTO_APPROVE (skip HITL panel)
// score 70–94  → SHOW HITL (green/amber confidence)
// score < 70   → SHOW HITL (amber/red, fields highlighted)
